# 智能适配模块设计方案

## 架构

现有引擎 = 匹配引擎 + 填充引擎 + 计算引擎
新增模块 = 结构分析器 + 自动匹配器 + 配置生成器

```
用户上传 → 结构分析器 → 自动匹配器 → 配置生成器 → 填充引擎 → 输出
            (模板分析)   (Excel分析)   (生成config)   (执行填充)
```

## 模块拆分

### 1. 模板结构分析器 (template_analyzer.py)
输入：Word 附注模板
输出：每张表的表名、列头、行标签列表、section 结构、合计行标记
方法：逐表扫描 + 表名识别 + 模式分类

### 2. Excel 结构分析器 (excel_profiler.py)
输入：决算套表 Excel
输出：每个 sheet 的表头行、数据行样例、列类型判断、会计科目列表
方法：模糊找表头 + 列类型推断 + 科目名提取

### 3. 智能映射器 (smart_matcher.py)
输入：模板结构 + Excel 结构
输出：MAPPINGS 配置（含置信度评分）
方法：表名匹配（3 级）+ 列模式匹配 + 行标签匹配 + 综合评分

### 4. 配置生成器 (config_generator.py)
输入：映射结果（含置信度）
输出：config/ 目录（YAML 文件）+ 差异报告
方法：序列化 MAPPINGS + 输出人工确认清单

### 5. 集成入口 (auto_adapt.py)
编排以上 4 个模块，一键执行

## 估算工期
- 模板分析器：1 天
- Excel 分析器：1 天
- 智能映射器：2 天
- 配置生成器：0.5 天
- 集成 + 测试：1 天
